### Reflection 
  On March 11th, I came to the lecture prepared to expand on my previous work by exploring the Drawing, Animation, and Simulation workspaces in Fusion 360. I learned that an assembly (the collection of all components in a model) is more than just a 3D shape. Rather, it is a dataset used for generating exploded views, assembly instructions, and Bills of Materials (BOMs).

  First, I created an exploded view animation, as this sequence could be reused for the technical drawing. The Animation workspace was straightforward, and I was quickly able to produce a video of my components separating.

  Next, I translated that animation into a technical drawing. I included a BOM in the top right corner along with callout balloons for each of the four components. Once the main assembly drawing was complete, I created individual technical drawings for each part. Fusion 360 made this process feel very intuitive.

  Lastly, the most difficult stage was preparing the model for the stress test simulation. Initially, the simulation failed because I hadn't correctly defined how the parts interacted. I realized that certain components needed to be excluded from the “study” to get accurate results. To resolve this, I followed a specific tutorial on “Manual Constraints.” The tutorial taught me how to physically simulate separate components pressing into one another. This step was crucial for making the simulation represent a real-world scenario rather than a theoretical one

  Overall, this weekly activity deepened my understanding of Fusion 360’s workflow. I realized how much time can be saved by using animations to create technical drawings and BOMs. Even though the stress test was a challenge, learning how it functioned made me feel more confident to handle future design projects in Fusion 360.
